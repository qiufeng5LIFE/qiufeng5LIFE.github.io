React中比较轻量化的状态管理

  • 组件间的状态共享只能通过将state提升到公共父级解决，层级太多组件树会很复杂，每一次更新都会重新渲染整个组件
  • context只能存储单一值，无法将状态分散，可能需要多个context

Recoil

  • 定义一个atom
  • 将RecoilRoot组件放置在需要使用atom的父组件
  • useRecoilState状态更新
  • useRecoilValue获取值

@umijs/plugin-model 一种基于hooks范式的简易数据管理方案

src/models目录下有 hooks model 时启用,文件名对应最终model的name

  • 定义model 路径必须为src/models/useTypeModel.ts
  • 使用

Hox

  • 定义一个store
  • 在需要引用的父节点加入InfoStoreProvider（自定义名称）节点
  • 在子节点中使用
  • 全局store

全局store用createGlobalStore替代createStore创建，只需要在应用最外层包裹HoxRoot就可以使用所有全局store

Umi useMode和Hox差异

  • useModel构建时动态在你的应用根节点之上再创建一个节点，新建useModel有目录位置的限制，启动时全部创建

  • hox是创建跟你应用根节点同级的节点，没有被调用（一般是没有进入相应的路由）就不会创建相应节点
